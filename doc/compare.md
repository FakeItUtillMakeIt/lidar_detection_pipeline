# 路径规划算法对比：势场法 vs A*

## 1. 算法概述

### 1.1 势场法 (Potential Field)

势场法是一种局部路径规划算法，通过模拟物理势场来引导车辆行驶：

- **引力场**：目标点对车辆产生吸引力，引力大小与距离成正比
- **斥力场**：障碍物和道路边界对车辆产生排斥力，斥力大小与距离成反比
- **合力**：车辆沿合力方向移动

```cpp
// 核心公式
F_attractive = attractive_gain * (goal - position)
F_repulsive = repulsive_gain * (1/distance - 1/influence_dist) / distance²
F_total = F_attractive + F_repulsive
```

### 1.2 A* 算法

A* 是一种全局路径规划算法，基于图搜索寻找最短路径：

- **网格化**：将规划空间离散化为网格
- **代价函数**：`f(n) = g(n) + h(n)`
  - `g(n)`：从起点到当前节点的实际代价
  - `h(n)`：从当前节点到终点的启发式代价（八边形距离）
- **搜索**：使用优先队列扩展代价最小的节点

## 2. 参数配置

### 2.1 势场法参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `attractive_gain` | 0.5 | 引力增益 |
| `repulsive_gain` | 50.0 | 斥力增益 |
| `obstacle_influence_dist` | 8.0m | 障碍物影响距离 |
| `step_size` | 0.5m | 梯度下降步长 |
| `max_iterations` | 200 | 最大迭代次数 |
| `convergence_threshold` | 0.1 | 收敛阈值 |

### 2.2 A* 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `grid_resolution` | 0.5m | 网格分辨率 |
| `max_grid_x` | 300 | x方向最大网格数 |
| `max_grid_y` | 50 | y方向最大网格数 |
| `obstacle_inflation` | 0.5m | 障碍物膨胀半径 |
| `use_diagonal` | true | 是否允许对角线移动 |

## 5. 优缺点分析

### 5.1 势场法

| 优点 | 缺点 |
|------|------|
| ✅ 计算速度快 (O(n)) | ❌ 容易陷入局部最优 |
| ✅ 轨迹平滑 | ❌ 障碍物密集时失败率高 |
| ✅ 实现简单 | ❌ 无法保证全局最优 |
| ✅ 适合局部避障 | ❌ 参数调节敏感 |

### 5.2 A* 算法

| 优点 | 缺点 |
|------|------|
| ✅ 能找到全局最优路径 | ❌ 计算量较大 (O(V log V)) |
| ✅ 避障能力强 | ❌ 轨迹网格化，不够平滑 |
| ✅ 成功率高 (80%) | ❌ 需要网格化空间 |
| ✅ 保证路径完整性 | ❌ 内存占用较大 |

## 7. 结论与建议

### 7.1 选择建议

| 场景 | 推荐算法 |
|------|----------|
| 障碍物稀疏 | 势场法 (快速) |
| 障碍物密集 | A* (可靠) |
| 实时性要求高 | 势场法 |
| 安全性要求高 | A* |
| 混合使用 | A*全局 + 势场法局部 |

### 7.2 未来优化方向

1. **混合规划器**：A* 生成全局路径，势场法进行局部避障
2. **D* 算法**：支持动态环境的增量式规划
3. **RRT/PRM**：采样-based 规划，适合高维空间
4. **深度强化学习**：端到端学习规划策略
